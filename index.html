Tools
Term What it actually means
Claude Chat Claude that you have conversations with — brainstorm, write, think out loud. Think of it like texting a really smart friend who never gets tired of your questions. Browser, mobile, desktop app
Claude CoWork Claude that can actually do things — touch your files, open your browser, build real deliverables. Think of it like having an assistant who can sit down at your computer and handle the task for you. Desktop app
Claude Code Claude that helps you write, edit, and understand code and files in your project. Think of it like having a developer sitting next to you, explaining everything in plain English. Desktop app · Terminal inside VS Code
Claude Chrome Extension Claude that travels with you while you browse — highlight text, ask questions, summarize pages without switching tabs. Think of it like having Claude look over your shoulder while you browse. Chrome
Visual Studio Code (VS Code) A text editor for viewing and editing code files — has a built-in Terminal so you don't need to open a separate one. Think of it like a home base for your project files — everything in one place, easy to navigate. Desktop
Terminal Where you type commands that do things — install stuff, create files, navigate folders. Think of it like a direct line to your computer — instead of clicking through menus, you type exactly what you want it to do. Inside VS Code · Standalone app
Typora A markdown editor that makes .md files easy to read and edit. Think of it like opening a markdown file in Word instead of Notepad — clean and readable instead of full of symbols. Desktop
How Claude Works
Term What it actually means
Prompt The instruction or question you give Claude to get a response. Think of it like placing an order — the more specific you are, the better what comes back. The better your prompt, the better your output.
Agent An AI that works on its own — give it a goal, it figures out the steps.
Workflow A series of steps that run in order, automatically.
Skill A saved set of instructions Claude follows every time — write it once, reuse forever. Works in Chat, CoWork, and Code.
Plugin A pre-built bundle of Skills + slash commands + tool connections. Think of it like hiring someone who already came trained — no setup required. Skills are the recipes you write yourself. Plugins are the cookbook a specialist hands you on day one. CoWork (cross-compatible with Code)

What's a Project?

A Project in Claude is a dedicated workspace where everything you need for a specific topic lives in one place — instructions, files, context. Every conversation inside that Project starts with Claude already knowing the background. No re-explaining from scratch. Think of it like a dedicated folder for a work project, except Claude automatically reads everything in it before you start talking.

What's a CLAUDE.md file?

A CLAUDE.md is a plain text file Claude reads automatically at the start of every session. It holds instructions you'd otherwise repeat every single time — your tone preferences, how you want things formatted, what to avoid, what tools to use. Think of it like an onboarding doc for a new hire — except the hire reads it perfectly every time.

What to put in it
  • Who you are and what you're working on
  • Your tone and style preferences
  • Tools you use
  • Things Claude should never do
  • Anything you keep having to re-explain
Where it lives
Personal (applies to all your projects) ~/.claude/CLAUDE.md
Project-specific (shared with a team) CLAUDE.md in the project folder
Tips
  • Keep it under 200 lines — shorter = better followed
  • Be specific: 'use bullet points' beats 'keep it clean'
  • Update it when Claude makes an assumption you want to correct — just say 'add this to my CLAUDE.md'
  • Run /init in Claude Code to auto-generate a starter file

Reusable instruction sets you write once and run any time. Each skill below includes a plain-English description and a copyable SKILL.md file you can drop into Claude Code.

apply-brand-voice

Acts as your brand voice guide — writes copy, reviews drafts for consistency, and answers messaging questions.

When to use: Any time you're creating or reviewing customer-facing content.
---
name: apply-brand-voice
description: Your company's positioning, messaging, and brand voice guide. Use when writing marketing copy, reviewing content for brand consistency, or answering questions about your positioning.
---

Serves as your single source of truth for positioning, messaging, and brand voice. Helps you write brand-aligned content, review drafts for consistency, and quickly reference your messaging framework.

<quick_start>
Choose what you need:
1. Write new copy
2. Review existing content
3. Look up messaging guidance
</quick_start>

<essential_principles>
Core Positioning: Define your category clearly. What problem do you solve and for whom?

Your Audiences: List the distinct audiences you serve and what each one cares about.

Tone Pillars (examples — replace with yours):
- Clear over clever — No jargon. Concrete outcomes.
- Educate, don't pitch — Lead with insight.
- Specific over vague — Show, don't tell.
</essential_principles>

What would you like to do?
1. Write copy — Generate brand-aligned content
2. Review content — Check a draft for brand consistency
3. Look up messaging — Answer questions about positioning or tone

<customize_this>
- Add your actual positioning statement and category definition
- Define your audience segments and what each cares about
- Add your tone pillars with real examples of good vs. bad
- Include your approved terminology and words to avoid
- Add your key differentiators and proof points
</customize_this>

apply-brand

Applies your brand styling to web projects — colors, typography, spacing, logos, light/dark themes.

When to use: When building or updating any web project and you want it to look on-brand automatically.
---
name: apply-brand
description: Applies your company's brand styling to web projects. Use when setting up design tokens, adding brand colors/typography, or migrating to your design system.
---

Integrate your company's design system into any web project. This skill provides guidance for CSS variables, typography, colors, spacing, and logo usage.

Your design system should provide:
- Color primitives and semantic colors
- Typography (fonts, sizes, weights)
- Spacing scale, border radius, shadows
- Light/dark theme support
- Logo assets and usage rules

<quick_start>
1. Install your design system package
2. Choose your integration method (CSS variables, Tailwind, JS/TS)
3. Test both light and dark modes
4. Validate logo usage against brand guidelines
</quick_start>

<essential_principles>
Semantic-First: Always prefer semantic tokens over raw color values.
Gradual Migration: When migrating existing projects, create an alias layer first.
Theme Support: Always test both light and dark modes.
</essential_principles>

<customize_this>
- Replace with your actual design token package and install instructions
- Add your color palette, font names, and spacing scale
- Add your logo usage rules and asset locations
- Define your light/dark theme switching mechanism
</customize_this>

prepare-preread-internal

Creates a confidential 7-section prep doc for your team before any external meeting — guest profile, objectives, talking points, landmines, questions, follow-up plan.

When to use: Before investor meetings, customer calls, partner conversations, or any meeting that benefits from shared team prep.
---
name: prepare-preread-internal
description: Create internal prereads — confidential strategy documents that prepare your team for external stakeholder meetings. Use when preparing for investor meetings, customer calls, partner conversations, or any external meeting that benefits from shared team prep.
---

Manages the full lifecycle of internal prereads — the confidential strategy documents your team uses to prepare for meetings with external stakeholders. These documents never leave your team.

<quick_start>
1. Create a folder for this contact
2. Copy your internal preread template
3. Research the guest thoroughly
4. Fill in all 7 sections
5. Circulate to your team 48 hours before the meeting
</quick_start>

<document_structure>
1. Guest Profile — Detailed bio, career arc, relevance to your company. Go beyond a LinkedIn summary.
2. Meeting Objectives — Primary goal (one sentence), secondary goals, and the explicit ask.
3. Talking Points — Your narrative tailored for this guest.
4. Relevant Materials — Papers, articles, prior interactions.
5. Landmines & Sensitivities — Minimum 2 specific, actionable entries.
6. Questions to Ask — Up to 5 questions, prioritized.
7. Follow-Up Plan — Action items with owner and concrete timeline.
</document_structure>

<writing_guidelines>
Be candid. This is internal only.
Be specific. Vague warnings don't help anyone.
Do your homework. The guest profile must show real research.
Landmines are the most valuable section. Anyone can Google a bio.
The ask must be honest — even 'no direct ask yet' should be stated explicitly.
Timelines must be concrete — not 'soon', use 'within 48 hours' or 'by Friday'.
</writing_guidelines>

<customize_this>
- Add your company's talking points and positioning narrative
- Add your template file location
- Define your folder/file naming convention
- Include a real example preread (sensitive info removed) for reference
</customize_this>

prepare-preread-external

Creates a polished 5-section doc to send to the person you're meeting — who's attending, what your company does, why their perspective matters, topics to discuss.

When to use: When you want to send a professional pre-read to an external stakeholder before a meeting.
---
name: prepare-preread-external
description: Create external prereads — polished documents sent to stakeholders before meetings. Use when preparing a document to share with an external contact before a call or meeting.
---

Creates polished, branded documents sent directly to external stakeholders before meetings. Shows the guest who they're meeting, what your company does, why their perspective matters, and what topics you'd like to discuss.

<quick_start>
1. Create a folder for this contact
2. Copy your external preread template
3. Fill in all 5 sections
4. Export to your sharing format
5. Send the shareable link to the guest
</quick_start>

<document_structure>
Keep it to 2 pages max — respect the guest's time.

1. Header — Title, meeting date, your logo
2. Attendees — Both sides, with names and roles
3. About [Your Company] / Since We Last Spoke — Tailored narrative.
4. Why We'd Value Your Perspective — Show you've done your homework.
5. Topics We'd Love to Discuss — 3 to 5 open-ended questions.

Never include internal strategy (objectives, the ask, landmines, follow-up plans). That belongs in the internal preread.
</document_structure>

<writing_guidelines>
Tailor the narrative to the guest — different audiences care about different things.
Show genuine research. Reference their actual work, not generic praise.
Keep it conversational — this is not a sales brochure.
Never write a preread that reads like a first meeting if it isn't one.
</writing_guidelines>

<customize_this>
- Add your company's one-liner and positioning narrative
- Add your logo and brand colors
- Define your folder/file naming convention
- Include a real example preread (sensitive info removed) for reference
</customize_this>

Technical Terms
Term What it actually means
API (Application Programming Interface) A way for two apps to talk to each other. Think of it like a waiter — you place your order, they go to the kitchen, and bring back the result. You probably won't need this until you're building something custom. Everywhere, behind the scenes
SDK (Software Development Kit) A pre-packaged toolkit developers use to connect Claude to their own apps. Think of it like IKEA instructions — everything you need, in one box. You probably won't need this until you're building something from scratch. For developers building custom tools
MCP (Model Context Protocol) A standard way to connect Claude to tools you already use — like Google Drive, Slack, or your calendar. Think of it like a universal adapter that lets Claude plug into your existing apps. CoWork and Code
